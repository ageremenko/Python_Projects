Результаты содержат:
1.  Скоры (вероятности дефолта/фрода) + бины (6 базовых переменных и доп.фичи) - проскоренные выборки в файле scorebin
- три PD скора (SCORE1, SCORE2, SCORE3) для оценки эффекта каждого из них, в т.ч. и с точки зрения фрода. Значения от 0 до 1, 0 – минимальная вероятность дефолта заемщиком, 1 – максимальная вероятность дефолта заемщиком.
- SCORE4 - Скор антифрод. Значения от 0 до 1, 0 – минимальная вероятность фрода заемщиком, 1 – максимальная вероятность фрода заемщиком.
- LIFETIME_BIN, ALL_CLC, PAY_MAX, BLOCK_DUR, BLOCK_CNT, CIRCLE - 6 базовых переменных, при их оценке дополнительно можно учитывать фрод составляющую, например в случае принятия значений "0" по всем указанным переменным высока вероятность фрода по заемщику.
- BIN1 - Идентификация фрод-трубки: Кол-во других сим-карт в бинарном виде вставленных в то же моб. устройство за последние 28 дней 
(0 - отсутствует информация, 
1 - 1 симкарта за 28 дней, 
2 - 2 симкарты за 28 дней, 
3 - 3 симкарты за 28 дней, 
4 - 4 и более симкарт за 28 дней)

2.  Оценка принадлежности номера тому, кто оформляет заявку, через валидацию даты рождения
Мы смотрим на того кто пользуется номером, а не на кого оформлена сим-карта, подобный подход позволяет исключить ошибки, возникающие в случаях, когда сим-карта оформлена на одного, а фактическим пользователем является другой.    
Выдаем  2 типа ответа - бинарный и вероятностный, один не получается из другого, т.к. строятся разными способами:
- бинарный (BIN2) - по вхождению в топ 5 дней (1 - вошел, 0 - не вошел, -1 - недостаточно информации) которые модель считает наиболее вероятным дату рождения




- вероятностный (SCORE5) - по результатам обучения модели, чем скор больше, тем больше вероятность соответствия указанной даты и даты рождения абонента (значения от 0 до 1, чем ближе к 1, тем выше вероятность совпадения ДР. В случае недостаточного количества данных для расчета скора возвращается -1)

3. Геовалидация - проскоренная выборка в файле geo
По  геовалидации есть две версии алгоритма, которые отличаются методикой выявления какой конкретно адрес считать домашним – соответственно эти версии v1 и v2 есть в названиях переменных. Также в названии переменных геовалидации есть дополнительная цифра, определяющая уровень точности в метрах:
is_home1000_v2 – валидация места жительства по сотам в радиусе 1000 метров, методика 2
По оценки качества геовалидации есть несколько вариантов:
- Как дополнительная переменная для использования во внутренней скоринговой модели. Результатом оценки является прирост Gini уточнённой модели по сравнению с исходной. Из прироста gini оценивается экономический эффект.
- Как правило отказа высокорисковым клиентам. На ретротесте можете оценить риск (и убыточность) тех клиентов, по которым геовалидация была неуспешной (ответ - 1). Сокращённый риск – и есть экономический эффект, который можно получить от использования отказа на основе геовалидации.
- Общим эффектом от геовалидации будет комбинация обоих пунктов.
